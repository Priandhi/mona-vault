## 1. Deep research (AI-Q pattern)

Kirim pertanyaan → server riset fan-out (search → fetch → verifikasi adversarial → sintesis) → report bersitasi. Bukan jawaban satu-shot; ada jejak sumber.

```python
from research_q import deep_research
job = deep_research("dampak EIP-4844 ke biaya L2 rollup 2024-2026", depth="thorough")
print(job.report)        # markdown + [1][2] citations
print(job.sources)       # list URL + klaim yang didukung
```

Kalau host punya skill harness `deep-research` (Claude/Cursor) → route ke situ. Kalau tidak → `research_q` jalan via `model_registry` + web fetch operator. **Setiap klaim penting wajib punya sitasi**; tanpa sumber → tandai "unverified", jangan dijual sebagai fakta (sejalan self-critique x5).

## 2. Scientific tool workflows (ToolUniverse-style)

Riset ilmiah = kombinasi banyak tool kecil, bukan satu model. Rangkai jadi pipeline:

```
QUERY → [literature search] → [fetch papers] → [extract method/result]
      → [cross-check claims] → [statistik/replikasi cek] → [synthesize + cite]
```

Pola: tiap langkah satu tool dengan output terstruktur, di-chain. Untuk domain spesifik (bio/kimia/material) sambungkan ke API/DB publik (PubMed, arXiv, Semantic Scholar, PubChem) — keyless atau free-tier. Verifikasi silang ≥2 sumber sebelum nyimpulin.

## 3. Hypothesis generation & experimental design

```
1. OBSERVASI  → apa yang diketahui? anomali/gap apa?
2. HYPOTHESIS → pernyataan testable + falsifiable (bukan "X mungkin bagus")
3. PREDICT    → kalau H benar, kita harus lihat Y di kondisi Z
4. DESIGN     → variabel kontrol vs uji, sample size, metrik, baseline, ablation
5. ANALYZE    → uji statistik yang cocok + threshold signifikansi DITENTUKAN DI DEPAN
```

Hipotesis harus falsifiable — kalau gak ada hasil yang bisa membantahnya, itu bukan hipotesis. Beda dari x6 (hypothesis buat *debug* sistem); m22 = hipotesis *riset/ilmiah*. Untuk eksperimen pada strategi/agent sendiri → pakai variance testing x5 sebagai alat ukurnya.

## 4. Scientific domain libraries & discovery

Untuk sains terapan, **jangan reimplementasi** — dispatch ke library/tool domain yang sudah ada & tervalidasi. Agent = orkestrator + verifikator, bukan sumber kebenaran biomedis.

| Domain | Tool/library matang (contoh) |
|---|---|
| Cancer genomics / RNA velocity | `scanpy`, `scvelo`, `anndata` |
| Drug-target binding / docking | `RDKit`, `AutoDock Vina`, `DiffDock` |
| Molecular dynamics | `OpenMM`, `GROMACS` |
| Geospatial science | `geopandas`, `rasterio`, `xarray` |
| Time-series forecasting | `statsmodels`, `prophet`, `darts` |

Pola: agent pilih tool yang tepat, siapin input, jalanin, **verifikasi hasil** (sanity check + sitasi metode), bukan ngarang angka. Library 100+ scientific = dispatch target, bukan klaim built-in.

### Hugging Science / Hub discovery
