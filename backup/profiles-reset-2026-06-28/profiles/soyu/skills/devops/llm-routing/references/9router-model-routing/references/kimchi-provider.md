## ⚠️ CRITICAL: Auto-Generated Keys Are INVALID

**Keys generated by Kimchidev.py script are REJECTED by Kimchi.dev** (HTTP 401). Kimchi detects automated bot signups and revokes keys — either immediately or shortly after generation.

**Symptoms:**
- Dashboard: "Invalid API key or base URL" for all auto-generated connections
- curl: `401 Authorization Required` (HTML response, not JSON error)
- Only manually-created keys work

**Root cause:** Kimchi.dev has bot detection that identifies:
- Automated browser patterns (Playwright/Selenium fingerprints)
- Disposable email domains (spamok.com)
- Rapid signup from same IP
- CAPTCHA solve patterns that are "too perfect"

**Verification test:**
```bash
# Test key directly through proxy
KEY=$(sqlite3 ~/.9router/db/data.sqlite "SELECT json_extract(data, '\$.apiKey') FROM providerConnections WHERE name='Kimchi-02';")
curl -s --proxy http://localhost:8888 --max-time 15 \
  -H "Authorization: Bearer *** \
  -H "Content-Type: application/json" \
  "https://llm.kimchi.dev/openai/v1/chat/completions" \
  -d '{"model":"minimax-m2.7","messages":[{"role":"user","content":"hi"}],"max_tokens":5}'

# 401 = key invalid. 200 = key works.
```

**Solution: Generate keys manually from HP browser** (different IP + human behavior):
1. Open https://app.kimchi.dev on phone browser (Chrome/Safari)
2. Sign up with real email (Gmail, Outlook, etc.)
3. Verify email
4. Go to Account → API Keys → Create
5. Copy key → add to 9Router via dashboard

**Do NOT trust script-generated keys.** Always test each key individually via curl before adding to 9Router.

The key was VALID; 9Router's dashboard test was unreliable. **Always verify with curl through proxy as ground truth, not dashboard test results.**

## Adding Keys to 9Router

### Dashboard UI (Most Reliable)
1. Dashboard → Providers → Kimchi → Add API Key
2. Paste key → Save
3. Restart: `sudo systemctl restart 9router`

### DB Insert (Works with correct format)
```python
import sqlite3, uuid, json
from datetime import datetime, timezone

db = sqlite3.connect(os.path.expanduser('~/.9router/db/data.sqlite'))
PROV_ID = db.execute("SELECT id FROM providerNodes WHERE name='Kimchi'").fetchone()[0]

conn_id = str(uuid.uuid4())
now = datetime.now(timezone.utc).strftime("%Y-%m-%dT%H:%M:%S.000Z")
data = json.dumps({
    "apiKey": "castai_v1_...",
    "testStatus": "pending",
    "providerSpecificData": {
        "baseUrl": "https://llm.kimchi.dev/openai/v1",
        "prefix": "kimchi",
        "apiType": "chat",
        "nodeName": "Kimchi-02",
        "connectionProxyEnabled": False,
        "connectionProxyUrl": "",
        "connectionNoProxy": ""
    },
    "defaultModel": "minimax-m2.7",
    "backoffLevel": 0,
    "lastError": None,
    "lastErrorAt": None
})
db.execute(
    "INSERT INTO providerConnections (id,provider,authType,name,priority,isActive,data,createdAt,updatedAt) VALUES (?,?,'apikey','Kimchi-02',1,1,?,?,?)",
    (conn_id, PROV_ID, data, now, now))
db.commit()
db.close()
# Then: sudo systemctl restart 9router
```

**CRITICAL:** Missing ANY of these fields causes errors:
- `baseUrl` → "Missing base URL"
- `prefix` → "The string did not match the expected pattern"
- `defaultModel` → routing issues

- Prefix: `castai_v1_`

## 9Router API Endpoints
- **Login:** `POST /api/auth/login` — `{"password":"..."}` → cookie auth
- **Provider Nodes:** `GET /api/provider-nodes`, `POST /api/provider-nodes`
- **Validate Key:** `POST /api/providers/validate` — `{"provider":"...", "apiKey":"..."}`
- **Connections:** `GET /api/providers`

**Note:** API key from `~/.9router/auth/cli-secret` does NOT work for dashboard API. Must use password login.
